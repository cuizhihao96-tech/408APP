# 408 考研刷题助手 — 实现方案

## 背景

用户提供四本王道 2027 教材 PDF，需要生成对应选择题+大题，支持按类型筛选练习。

## PDF 现状

- PDF 是高清扫描版（图片层，无文字层），无法直接提取文本
- 已成功提取书签/目录结构（PyMuPDF）
- 四本书合计 ~1400 页、~200 个知识节点

## 实施策略

**两阶段：**
1. 先搭建完整 App 框架（Electron + HTML/CSS/JS），与背词助手同架构
2. 基于 408 考纲知识点，按教材章节结构批量生成题目数据

## 题目数据设计

### JSON 格式（`data/questions.json`）

```json
[
  {
    "id": "os-001",
    "subject": "操作系统",
    "chapter": "1.1 操作系统的基本概念",
    "type": "choice",
    "question": "操作系统最基本的特征是？",
    "options": ["并发和共享", "虚拟和异步", "共享和虚拟", "并发和异步"],
    "answer": 0,
    "explanation": "并发和共享是操作系统两个最基本的特征，二者互为存在条件。"
  },
  {
    "id": "ds-001",
    "subject": "数据结构",
    "chapter": "1.2 算法和算法评价",
    "type": "big",
    "question": "分析以下代码的时间复杂度：\nfor(i=1; i<=n; i*=2)\n  for(j=1; j<=i; j++)\n    x++;",
    "answer": "O(n)\n外层循环 O(logn)，内层循环 i 次。总执行次数为 1+2+4+...+2^k ≈ 2n-1，故时间复杂度为 O(n)。",
    "explanation": ""
  }
]
```

### 字段说明

| 字段 | 说明 |
|---|---|
| id | 唯一标识，格式：科目缩写-序号 |
| subject | 操作系统 / 数据结构 / 计算机组成原理 / 计算机网络 |
| chapter | 对应教材章节 |
| type | choice（选择题）/ big（大题） |
| question | 题干 |
| options | 仅选择题，4 个选项的数组 |
| answer | 选择题填正确索引(0-3)，大题填答案文本 |
| explanation | 解析（选择题必填，大题可选） |

## App 页面结构

### 首页
- 四科卡片网格（2×2），每科显示进度（已做/总量）
- 今日统计（已刷题数、正确率）
- 底部：随机刷题、错题本按钮
- 顶部：练习模式切换（仅选择 / 仅大题 / 混合）

### 答题页
- 顶部：返回 + 科目章节 + 进度
- 选择题：题干 + 四选项，点击即判 + 显示解析
- 大题：题干 + 「显示答案」按钮，展开后自判（我对了/我错了）

### 错题本
- 按科目筛选，显示做错的题
- 点开可重做
